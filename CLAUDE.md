# CLAUDE.md — Employee Retention Risk Project

## Project Overview

A machine learning pipeline for predicting employee attrition risk using XGBoost, SHAP explainability, and probability calibration. The goal is to help HR teams identify at-risk employees and understand the key drivers behind potential departures.

## ML Pipeline Summary (`retention_pipeline.py`)

1. `load_data()` — reads preprocessed Excel data
2. `engineer_features()` — creates derived features (Engagement Index, Promotion Rate, Overtime × Role, polynomial interactions)
3. `select_features()` — variance threshold → correlation filter → RFE (top 20 features)
4. SMOTE resampling to address class imbalance
5. `train_model()` — XGBoost + GridSearchCV + Platt scaling calibration
6. `evaluate_model()` — classification report, confusion matrix, AUC-ROC, precision-recall curve
7. `find_best_threshold()` — optimizes decision threshold for F1
8. `assign_risk()` — buckets employees into Low / Moderate / High risk tiers
9. `explain_employee()` — per-employee SHAP explanation

## Development Notes

- The `np.int` compatibility shim in `retention_pipeline.py` (line 29–30) handles a numpy deprecation for older versions of SHAP/XGBoost.
- Feature selection uses aggressive filtering (variance → correlation → RFE) to avoid overfitting on the small IBM dataset (~1,470 rows).
- Risk tiering uses the 90th percentile as the High/Moderate boundary, making it threshold-adaptive.
- The calibrated model uses Platt scaling (`method='sigmoid'`) with 5-fold cross-validation.
